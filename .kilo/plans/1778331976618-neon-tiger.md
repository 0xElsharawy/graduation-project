# Chat Feature Implementation Plan

## Overview

Build a real-time workspace chat feature for the project management application, enabling direct messaging (DMs) between workspace members with file sharing, emoji reactions, and workspace lead moderation capabilities.

## Architecture

### Backend (NestJS)
- **Database Schema**: New `chat` module with tables for conversations, messages, and participants
- **Real-time**: WebSocket gateway for live messaging (or polling fallback)
- **REST API**: CRUD endpoints for chat resources

### Frontend (Next.js)
- **Chat Page**: New `/[workspace]/chat` route with responsive layout
- **Components**: Conversation sidebar, message list, message composer, user presence
- **State**: React Query for server state, Jotai for UI state

## Implementation Steps

### Phase 1: Database Schema & Backend

#### 1.1 Create Chat Database Schema (`apps/api/src/db/schema/chat.ts`)
```typescript
// conversations table
export const dmConversations = pgTable('dm_conversations', {
  id: uuid('id').primaryKey().defaultRandom(),
  workspaceId: uuid('workspace_id').references(() => workspaces.id, { onDelete: 'cascade' }).notNull(),
  user1Id: text('user1_id').references(() => users.id, { onDelete: 'cascade' }).notNull(),
  user2Id: text('user2_id').references(() => users.id, { onDelete: 'cascade' }).notNull(),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').onUpdate(() => new Date()).notNull(),
}, (t) => [
  unique('dm_unique_pair').on(t.workspaceId, t.user1Id, t.user2Id),
]);

// messages table
export const chatMessages = pgTable('chat_messages', {
  id: uuid('id').primaryKey().defaultRandom(),
  conversationId: uuid('conversation_id').references(() => dmConversations.id, { onDelete: 'cascade' }).notNull(),
  senderId: text('sender_id').references(() => users.id, { onDelete: 'set null' }),
  content: text('content'), // nullable for file-only messages
  fileUrl: text('file_url'),
  fileType: text('file_type'),
  fileSize: integer('file_size'),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { withTimezone: true }),
}, (t) => [
  index('chat_messages_conversation_created_idx').on(t.conversationId, t.createdAt),
]);

// message reads tracking
export const messageReads = pgTable('message_reads', {
  id: uuid('id').primaryKey().defaultRandom(),
  messageId: uuid('message_id').references(() => chatMessages.id, { onDelete: 'cascade' }).notNull(),
  userId: text('user_id').references(() => users.id, { onDelete: 'cascade' }).notNull(),
  readAt: timestamp('read_at', { withTimezone: true }).defaultNow().notNull(),
}, (t) => [
  unique('message_user_unique').on(t.messageId, t.userId),
]);

// message reactions
export const messageReactions = pgTable('message_reactions', {
  id: uuid('id').primaryKey().defaultRandom(),
  messageId: uuid('message_id').references(() => chatMessages.id, { onDelete: 'cascade' }).notNull(),
  userId: text('user_id').references(() => users.id, { onDelete: 'cascade' }).notNull(),
  emoji: text('emoji').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
}, (t) => [
  unique('reaction_unique').on(t.messageId, t.userId, t.emoji),
]);
```

#### 1.3 API Endpoints (DMs Only)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/workspaces/:id/chat/conversations` | List DMs for current user |
| POST | `/workspaces/:id/chat/conversations` | Create DM with another member |
| GET | `/workspaces/:id/chat/conversations/:id/messages` | List messages in DM |
| POST | `/workspaces/:id/chat/conversations/:id/messages` | Send message |
| PATCH | `/workspaces/:id/chat/messages/:id` | Edit own message |
| DELETE | `/workspaces/:id/chat/messages/:id` | Delete own or others' message (workspace leads) |
| POST | `/workspaces/:id/chat/messages/:id/reactions` | Add reaction |
| POST | `/workspaces/:id/chat/messages/:id/attachments` | Upload file attachment |

## Scope Clarifications (User Confirmed)

Based on user responses:
- **DMs only** - No public channels; only direct messaging between workspace members
- **No threading** - Simple linear conversation flow
- **Full file upload support** - Images, documents, and other files
- **Both WebSocket + Polling** - Real-time with fallback
- **Workspace lead controls** - Admins can moderate (delete others' messages, manage DMs)

## Dependencies to Add

- `emoji-picker-react` - For emoji picker in composer
- Socket.io client - For frontend real-time connection

1. **Backend Schema & API** - Start with database schema and REST endpoints
2. **Basic Frontend** - Create chat page with conversation/message components
3. **Real-time Layer** - Add WebSocket for live updates
4. **File Upload** - Integrate file attachment support
5. **Polish** - Add reactions, typing indicators, read receipts
